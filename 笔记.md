# Terminal Helper 项目笔记

## 项目结构
```
Terminal_helper/
├── data/
│   └── sample.json                 # 示例数据
├── src/
│   ├── main/                       # Electron 主进程
│   │   ├── index.js
│   │   ├── ipcHandlers.js           # IPC：执行命令 / 导入导出 / 运行日志等
│   │   └── store.js                 # JSON 数据持久化
│   ├── preload/                    # 桥接层
│   │   └── index.js                 # 暴露给前端的 API
│   ├── renderer/                   # 前端界面 (Vue 3)
│   │   ├── App.vue
│   │   ├── main.js
│   │   ├── style.css
│   │   ├── index.html
│   │   ├── assets/
│   │   ├── components/
│   │   └── modules/
│   │       ├── cards/
│   │       ├── collections/
│   │       └── terminal/terminalStore.js
│   └── stores/                     # 预留/历史目录
├── package.json
├── vite.config.js
├── tailwind.config.cjs
└── postcss.config.cjs
```

数据文件实际落地位置：
- `%APPDATA%/terminal-helper/data/terminal-helper.json`（Windows 对应 `C:\Users\<user>\AppData\Roaming\terminal-helper\data\terminal-helper.json`）
- 首次启动会从 `data/sample.json` 初始化

## 技术栈
- Electron 41（桌面端壳）
- Vite 8 + @vitejs/plugin-vue（构建）
- Vue 3（渲染层）
- Pinia（状态管理）
- Tailwind CSS（样式）
- lucide-vue-next（图标）
- vuedraggable（拖拽排序能力）
- Node.js child_process（命令执行）
- iconv-lite（Windows 输出编码解码）

## 项目核心功能
- 可视化卡片式终端命令管理
- 卡片集合管理与串行执行
- 日志面板实时输出与失败定位
- 本地 JSON 持久化与导入导出
- 可自定义 CMD/PowerShell/Bash 路径

## 目前已有功能
- 卡片管理
  - 新建 / 编辑 / 复制 / 删除
  - 搜索（命令 / 描述 / 关键词）
  - 运行终端选择（CMD / PS / Bash）
- 集合管理
  - 新建 / 编辑 / 复制 / 删除
  - 集合内支持重复卡片
  - 集合顺序可编辑（运行顺序数字气泡）
  - 集合列表支持搜索
  - 集合管理页支持导入/导出
  - 集合列表区域独立滚动
- 执行与日志
  - 单卡执行与集合串行执行（同一终端会话）
  - 日志面板显示 stdout/stderr、状态、耗时
  - 运行中可停止（按钮变为红色停止图标）
  - 集合详情固定“停止”卡片
  - 可配置停止命令（终端类型 / 是否新终端执行）
  - 停止支持进程树终止
- 导入导出
  - 全量导入/导出 JSON
  - 集合内导入/导出（仅集合所含卡片）
- 设置
  - CMD / PowerShell / Bash 路径设置
  - 路径检测与应用
- UI
  - 3 列固定网格卡片
  - 底部浮动操作按钮
  - 抽屉式新建/编辑表单
  - 日志面板可折叠
